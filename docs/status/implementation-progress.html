FreeCAD MCP Workbench Implementation Progress

Updated May 24, 2026. Scope: roadmap phases 1 through 5 from docs/design/implementation-roadmap.md.

PhaseStatusImplementedRemaining Validation
Phase 1: Live Server Proof of Concept Implemented, needs FreeCAD runtime validation Localhost Streamable HTTP adapter, MCP SDK dependency check, lifecycle controller, logging, read and mutating tool handlers. Verify official MCP SDK startup inside FreeCAD's bundled Python and confirm GUI-thread dispatch behavior with a live FreeCAD GUI.
Phase 2: Workbench Packaging Implemented InitGui.py, Workbench registration, toolbar/menu commands, dependency path support, status reporting, log command, copy URL command. Confirm Workbench discovery from the target FreeCAD user Mod directory on macOS, Windows, and Linux.
Phase 3: V1 Tool Implementation Implemented with conservative FreeCAD API coverage All v1 tool names are registered. Handlers validate inputs, return structured results, avoid arbitrary Python execution, and cover inspection, document control, primitives, sketch workflow, property editing, fillet, STEP export, and STL export. Run against real FreeCAD to tune Part Design body activation, selected-face sketch attachment, constraint signatures, and export module behavior across FreeCAD versions.
Phase 4: Testing Automated unit tests implemented Standard-library tests with mocked FreeCAD modules cover schemas, validation errors, primitive creation, recompute calls, property allowlisting, sketch geometry identifiers, save suffix validation, and port conflict fallback. Add live integration tests once a supported FreeCAD executable path is configured. Complete a manual acceptance run in the GUI.
Phase 5: Polish and Hardening Initial hardening implemented Port fallback, clearer dependency flow, local logs, status messages, install/troubleshooting docs, explicit progress tracking, and object summaries with transforms and bounding boxes where available. Improve status UI beyond dialogs/console messages after live Workbench testing. Add richer recompute recovery hints from real FreeCAD exception types.

Key Files

  • InitGui.py: FreeCAD Workbench entrypoint.
  • freecad_mcp_workbench/controller.py: server lifecycle and status.
  • freecad_mcp_workbench/server.py: MCP Streamable HTTP runtime.
  • freecad_mcp_workbench/tools.py: v1 tool handlers.
  • freecad_mcp_workbench/schemas.py: tool input schema registry.
  • tests/: mocked FreeCAD unit tests.
